# Проект Tetr 

## О проекте
**Tetr** — это клон классической игры «Тетрис», разработанный на игровом движке **Godot 4**. Проект реализует базовые механики Тетриса, включая правильное вращение фигур с использованием Super Rotation System (SRS), а также систему "призрачной" фигуры (Ghost Piece).
В ходе доработки в проект были внедрены современные игровые механики, полноценный UI, система очков и сохранение прогресса, что делает его завершенным для портфолио.

## Структура проекта

### 1. Ресурсы (Resources)
В папке Resources/ хранятся конфигурационные файлы *_piece_data.tres для каждого типа тетрамино (I, J, L, O, S, T, Z). Они базируются на классе piece_data.gd и содержат текстуру фигуры, её тип и позицию спавна.

### 2. Сцены (scenes)
- main_menu.tscn — главное стартовое меню игры.
- mode_select.tscn — окно выбора режима игры (Classic / Arcade) с отображением сохраненного рекорда (High Score).
- main.tscn — основная сцена игры, содержащая игровое поле, спавнер и интерфейс (UI) текущего состояния (Счет, Уровень, Линии, экран Game Over).
- 	etromino.tscn — сцена падающей фигуры (состоит из 4 блоков).
- piece.tscn — отдельный компонент из которых состоят фигуры.
- ghost_tetromino.tscn — полупрозрачная проекция падающей фигуры в самом низу.

### 3. Скрипты (scripts)
В директории scripts/ находится вся логика игры:

* main.gd — Главный контроллер сцены main.tscn. Обновляет UI во время игры и вызывает экран поражения.
* shared.gd — Глобальный синглтон (Autoload). Хранит константы, словари координат блоков, матрицы вращения, таблицы wall kicks, а также **менеджер сохранений** (запись рекорда в локальный файл).
* oard.gd — Управляет сеткой игрового поля. Осуществляет поиск и удаление заполненных линий, математику начисления очков (по Nintendo Scoring System), отслеживает Game Over.
* piece_spawner.gd — Отвечает за генерацию фигур с использованием **7-bag randomizer** (алгоритм "мешка"), обеспечивая равномерное выпадение всех тетрамино.
* 	etromino.gd — Логика управления фигурой: передвижение, вращение, отталкивание от стен и проверка на столкновения прямо на спавне (смерть).
* piece.gd / ghost_tetromino.gd / piece_data.gd — вспомогательные скрипты для блоков и ресурсов.

## Оценка проекта (Ревью)

**Плюсы (Что реализовано):**
1. **Полноценный Game Loop:** Реализовано главное меню, начисление очков, повышение скорости (через уровни), состояние Game Over и локальное сохранение лучшего счета (High Score).
2. **Достоверная механика:** Работает Super Rotation System, Ghost piece и честный алгоритм "мешка" для рандомизации фигур, как в современных версиях тетриса.
3. **Архитектура:** Чистое разделение ответственности между компонентами (Board считает поле, Tetromino отвечает за ввод и вращения, Main управляет UI, Shared - глобальными данными).
4. **Data-driven подход:** Грамотное применение Custom Resources движка Godot для свойств фигур вместо хардкода.

**Зоны для улучшения на будущее:**
1. **Остаточные Magic Numbers:** В 	etromino.gd всё ещё присутствуют захардкоженные границы экрана (ounds). Было бы полезно перевести их расчет динамически под размеры окна и сетки.
2. **Коллизии:** На данный момент часть коллизий полагается на перебор позиций дочерних объектов и pieces, полный переход на строгую двумерную математическую сетку (Grid) для обработки препятствий укрепил бы точность движка при высоких скоростях.
3. **Реализация Arcade Mode:** Подготовка и написание отдельной логики для новых бонусных режимов в игре.
